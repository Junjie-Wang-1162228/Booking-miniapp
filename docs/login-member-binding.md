# 微信登录与会员绑定说明

本文说明真实上线时“小程序微信账号”和“拳馆会员档案”的关系。

## 核心结论

- 微信登录只负责确认“这是哪个微信用户”。
- 会员档案才承载真实业务数据：姓名、手机号、会员号、所属门店、课时余额、预约、消课记录。
- `WechatAccount` 是绑定表，用 `(AppID, openid)` 绑定到一个会员 `User`。
- 不把 `openid` 当成长期业务身份。换小程序主体或 AppID 后，`openid` 可能变化；长期应以内部会员档案、手机号、会员号为准。

## 上线推荐流程

生产或真实试运营建议关闭自动开卡：

```text
WECHAT_AUTO_PROVISION_ENABLED="false"
```

流程：

1. 会员第一次打开小程序，前端调用 `wx.login`。
2. 后端用 `code2Session` 换到当前 AppID 下的 `openid`。
3. 如果这个 `openid` 已绑定会员，后端返回 JWT，会员进入约课。
4. 如果未绑定，后端生成 6 位绑定码并返回给小程序。
5. 小程序提示会员把绑定码发给拳馆管理员。
6. 管理员在 Web 后台创建或选择会员档案，输入绑定码完成绑定。
7. 会员重新进入小程序后，进入自己的会员档案，只能看到自己所属门店、自己的课时、预约和消课记录。

## 换微信或绑定错账号

如果会员换微信，或管理员把 openid/绑定码绑到了错误会员：

1. 管理员在 Web 后台找到错误会员。
2. 点击“解绑微信”，确认后解除当前 AppID 下的微信账号绑定。
3. 让会员重新打开小程序获取 6 位绑定码，或由管理员录入新的 openid。
4. 管理员在正确会员档案上点击“绑定微信”，输入绑定码或 openid 完成重绑。

绑定会写入 `WECHAT_BIND` 审计日志；解绑会写入 `WECHAT_UNBIND` 审计日志，便于追溯谁在什么时候处理了账号绑定。

## 换手机号或会员号纠错

手机号、姓名、会员号属于会员档案资料，不影响已经绑定的微信 openid。管理员可在 Web 后台会员列表中点击“编辑资料”修改：

- 姓名
- 手机号
- 会员号

后台会校验手机号不能和另一个会员重复。更新会写入 `MEMBER_UPDATE` 审计日志，便于追溯资料变更。

## 内测自动开卡流程

内部一次性测试可以打开自动开卡：

```text
WECHAT_AUTO_PROVISION_ENABLED="true"
WECHAT_AUTO_PROVISION_BRANCH_NAME="城东店"
WECHAT_AUTO_PROVISION_LESSONS="10"
```

此时未知微信账号首次登录会自动创建测试会员：

- 昵称：`微信测试会员-<openid后6位>`
- 默认门店：配置的门店，或第一个启用门店
- 初始课时：配置的课时数

这个模式只适合开发和内测，不建议对真实会员开放。

## 数据隔离规则

- 会员端所有课程、预约、课时、消课查询都从 JWT 当前用户推导，不信任前端传入的 `userId`。
- 会员只能访问自己 `MemberBranch` 中的门店。
- 课时余额按 `userId + branchId` 隔离。
- 管理员消课受门店权限控制，店长不能操作无权限门店。
- 同一个 `(AppID, openid)` 只能绑定一个会员，避免一个微信串到多个会员档案。

## 当前已覆盖的自动化验证

API E2E 已覆盖：

- 已绑定微信登录到正确会员。
- 未绑定微信在自动开卡关闭时返回 6 位绑定码，不创建会员。
- 管理员用绑定码把微信绑定到已有会员。
- 管理员可以解绑错绑微信，并把同一个 openid 重绑到另一个会员。
- 管理员可以编辑会员姓名、手机号、会员号，并拒绝重复手机号。
- 自动开卡开启时，未知微信创建测试会员、分配默认门店和初始课时。
- 微信测试会员完成约课、生成提醒任务、后台消课、课时余额减少。
- 管理员门店权限、重复消课、会员数据隔离。

## 手工验收路径

1. 后端 `.env` 设置真实 `MINIAPP_APP_ID` 和 `MINIAPP_APP_SECRET`。
2. 设置 `WECHAT_AUTO_PROVISION_ENABLED="false"`。
3. 启动 API、后台和小程序构建。
4. 用测试微信账号打开小程序，看到 6 位绑定码。
5. 后台创建会员，输入绑定码绑定。
6. 重新进入小程序，确认显示该会员姓名、门店和课时。
7. 预约课程，后台消课，确认账户页课时减少并出现消课记录。
