# 数据保留与删除策略

最后更新：2026-06-13

本文档是拳馆约课小程序的上线材料底稿，用于明确当前系统收集哪些业务数据、为什么收集、保留多久、如何处理删除或纠错请求。它不是法律意见；正式上线前应由拳馆主体按实际运营地区、合同、隐私政策和微信后台隐私保护指引确认。

## 数据分类

| 数据类型 | 当前字段/模型 | 用途 | 首发必要性 |
| --- | --- | --- | --- |
| 会员身份资料 | `User.displayName`、`User.phone`、`MemberBranch.memberNo` | 建立会员档案、前台核对身份、后台搜索会员、绑定课时 | 必要 |
| 微信身份绑定 | `WechatAccount.appId`、`openid`、`unionid`、`WechatBindingTicket.code` | 微信登录、把微信账号绑定到会员档案、防止账号串绑 | 必要 |
| 门店与权限 | `Gym`、`Branch`、`MemberBranch`、`StaffBranchAssignment` | 控制会员可见门店、店长/管理员操作边界 | 必要 |
| 课程与预约 | `BoxingClass`、`Booking` | 展示可预约课程、记录预约、取消和到课状态 | 必要 |
| 课时与消课 | `LessonBalance`、`LessonBalanceAdjustment`、`LessonDeduction` | 记录课时余额、调整原因、消课流水和操作人 | 必要 |
| 通知任务 | `NotificationJob`、`NotificationLog` | 发送开课提醒、记录发送结果、支持失败排查 | 必要，仅在启用订阅消息时使用 |
| 审计日志 | `AuditLog` | 追溯后台创建课程、取消课程、消课、课时调整、会员资料修改、微信解绑等关键操作 | 必要 |
| 管理员账号 | `User.username`、`passwordHash`、`StaffBranchAssignment.role` | 后台登录和权限控制 | 必要 |

当前首发范围不应收集身份证号、出生日期、精确定位、通讯录、相册、麦克风、摄像头、生物识别、支付卡号等与拳馆约课无关的数据。未来如接入支付、位置导航、上传图片或客服工单，应先更新隐私政策、微信后台隐私保护指引和本策略。

## 使用说明

- 手机号：用于会员档案识别、后台搜索、前台联系和纠错，不作为公开展示内容。
- 微信 `openid`：用于当前 AppID 下的微信登录绑定，不作为长期业务身份。换主体或换 AppID 时，应以会员档案、手机号或会员号完成重绑。
- 预约记录：用于会员查看本人预约、后台排课和消课，不向其他会员展示。
- 课时记录：用于计算余额、消课和纠错，会员只能查看本人记录，后台按门店权限查看。
- 消课记录：用于证明课时消耗和运营对账，不允许硬删除；错误消课通过课时调整或冲正说明处理。
- 审计日志：用于安全和财务追溯，只面向有权限的后台人员。

## 保留周期

| 数据类型 | 建议保留周期 | 删除或归档方式 |
| --- | --- | --- |
| 会员身份资料 | 会员关系结束后至少 2 年，或按合同/当地法规要求更长时间 | 可停用会员账号；确需删除时先确认无未结清课时、未完成纠纷和未完成退款 |
| 微信绑定关系 | 会员解绑后立即删除对应 `WechatAccount`；绑定码过期或绑定完成后不再用于登录 | 后台解绑/重绑流程处理 |
| 预约记录 | 课程结束或取消后至少 2 年 | 不硬删；用于纠纷处理、运营统计和安全审计 |
| 课时余额和调整流水 | 会员课包有效期结束后至少 2 年 | 不硬删流水；余额纠错走调整记录 |
| 消课记录 | 至少 3 年 | 不硬删；错误记录通过冲正/调整说明处理 |
| 通知任务和发送日志 | 至少 180 天 | 可按批处理归档或删除过期日志，保留失败原因用于排障 |
| 审计日志 | 至少 3 年 | 不向普通后台提供删除能力 |
| 数据库备份 | 至少 14 天；发版前备份保留到该版本稳定后 | 按 `docs/production-data-runbook.md` 的备份策略加密保存和定期清理 |

保留周期是默认建议。上线前应由拳馆主体确认是否需要按当地法规、合同、财务和争议处理要求调整。

## 删除与纠错流程

会员提出删除或纠错请求时，后台操作人员应按以下流程处理：

1. 核验请求人身份，例如通过会员手机号、会员号、微信绑定状态或线下身份确认。
2. 判断请求类型：资料更正、微信解绑、停用会员、删除可删除数据、导出本人记录。
3. 资料更正：在后台编辑姓名、手机号、会员号，并保留 `MEMBER_UPDATE` 审计日志。
4. 微信解绑：通过后台解绑微信账号，并保留 `WECHAT_UNBIND` 审计日志。
5. 预约或消课纠错：不直接删除流水，通过取消预约、课时调整或备注说明保留可追溯记录。
6. 删除请求：先确认无未完成课程、未结清课时、退款、争议或法定保留义务，再执行最小化删除或停用。
7. 备份中的历史数据不单独改写；按备份保留周期自然过期。若从备份恢复，应重新执行删除/纠错请求记录中的必要操作。

## 上线检查

- 微信后台隐私保护指引应覆盖：手机号、微信 `openid` 或微信用户标识、预约记录、课时/消课记录、通知订阅相关信息。
- 小程序账户页已提供隐私政策入口和约课规则入口；正式上线前应按拳馆主体、实际服务范围和微信后台配置复核文案。
- API 日志、错误响应和前端错误提示不得输出手机号、`openid`、JWT、AppSecret、数据库连接串、密码或密码哈希。
- 生产数据库和备份应限制访问权限，并记录谁在何时导出或恢复数据。
- 新增任何数据字段、第三方服务、支付能力、位置能力或上传能力前，先更新本文档和隐私材料。
